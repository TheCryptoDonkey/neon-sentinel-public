# Neon Sentinel Release Notes

## DNS

- `neonsentinel.com` is the production game host.
- `app.neonsentinel.com` is also served by the production game vhost.
- `www.neonsentinel.com` should resolve to the same origin and is served by the same Caddy vhost.
- `neonsentinel.playechoseven.com` is an `A` record to `95.217.39.110`, DNS-only.
- `playechoseven.com` is an `A` record to `95.217.39.110`, DNS-only.
- `www.playechoseven.com` is a proxied CNAME to `playechoseven.com`.

Use `neonsentinel.com` for the public game host. Keep `neonsentinel.playechoseven.com` available as the staging/smoke-test host.

## Shared Hetzner Host

The Hetzner box already runs other games, so Neon Sentinel should deploy as a separate app:

- Dedicated directory: `/opt/neonsentinel`.
- Dedicated static service/container and vhost.
- No changes to existing game compose files, web roots, ports, or certs unless audited first.
- Before deployment, run a read-only audit of active containers, web server config, occupied ports, and current certificate setup.
